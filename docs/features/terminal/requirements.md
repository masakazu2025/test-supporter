# 端末 要件

## 概要

リモート端末のファイルを検索・閲覧・採取する機能。
採取インフラ（プール）の上に乗る操作UIとして機能する。

## 操作の種類

| 操作 | 概要 | データソース |
|------|------|------------|
| **検索** | エントリ・画像・ログをキーワード・条件で検索 | プール優先 → 端末直接（透過） |
| **閲覧** | ファイルの内容を表示・フォーマット | プール優先 → 端末直接（透過） |
| **採取** | ファイルをプールに取得する | 端末直接 → プール |

## 閲覧対象

エントリ・画像・ログはそれぞれ独立した閲覧画面を持つ。

| 種別 | 説明 | 画面イメージ |
|------|------|------------|
| `エントリ` | テスト実施の1件分の活動記録（アーカイブ） | 一覧 + 詳細展開 |
| `画像` | スクリーンショット | サムネイル一覧 + 拡大表示 |
| `ログ` | 端末ログ（将来構想） | テキストスクロール |

> 各画面の詳細レイアウトはスケルトンフェーズで確認する。

## 採取（採取機能との関係）

採取の設計・実装は [docs/features/collection/](../collection/) を参照。
端末機能はその操作UIを提供する。

| 採取方法 | トリガー |
|---------|---------|
| 自動採取 | 監視開始/停止ボタン |
| 指定採取 | エントリNo・日時範囲などを指定してボタン操作 |

## 検索コンポーネントの共用

エントリ検索コンポーネントは**評価機能のエビデンス選択でも共用**する。

```
端末タブ: [検索条件] → [検索結果一覧] → 閲覧・採取操作
評価タブ: [検索条件] → [検索結果一覧] → エビデンスとして選択
```

コンポーネントは同一だが、選択後のアクションが異なる（表示 vs 紐付け）。

## データソースの透過性

ユーザーはプール・端末直接を意識しない。
バックエンドのファイル解決レイヤーが透過的に処理する。

```
検索・閲覧リクエスト
  → プールを検索
      ヒット → プールから返す
      ミス   → 端末直接アクセス → 返す
```
