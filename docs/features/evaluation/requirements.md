# 評価 要件

## 概要

テスト仕様書（Excel）と紐づけてエビデンスを管理し、不具合を記録する機能。
採取したファイル（プール or 端末直接）をテストケースに結びつける。

## テストケース管理

テスト仕様書のフォーマットが外部依存のため、最小限の情報のみ保持する。

```python
class TestCase:
    spec_ref: str        # テスト仕様書上の識別子（項番・ケース名など自由記述）
    name: str | None     # 補足用の名称（任意）
    evidences: list      # エビデンスのリスト
```

- `spec_ref` は自由文字列（仕様書フォーマットが決まったときに拡張できる余地を残す）
- テストケースの登録・編集・削除はユーザーが手動で行う

## エビデンス

テストケースに紐づくファイル（エントリ・画像など）。

- ファイルはプールまたは端末直接から取得する（ユーザーは意識しない）
- エビデンスの追加は「エントリ検索コンポーネント」経由（端末機能と共用）
- 複数ファイルを1テストケースに紐付け可能

## 不具合管理

| 項目 | 説明 |
|------|------|
| テストケースへの紐付け | 任意（`test_case_id` は nullable） |
| 単独起票 | テストケースと関係なく起票可能 |
| 資料 | ファイル（エントリ・画像など）を添付可能 |
| 内容記載 | 不具合の説明・再現手順などをテキストで記録 |

```python
class Defect:
    test_case_id: str | None  # nullable → 単独起票可
    title: str
    description: str
    materials: list           # 添付ファイルのリスト
```

## ユースケース

| # | 操作 | 説明 |
|---|------|------|
| 1 | テストケース一覧表示 | 登録済みテストケースを一覧で確認 |
| 2 | テストケース登録 | spec_ref と name を入力して登録 |
| 3 | エビデンス追加 | エントリ検索でファイルを選び、テストケースに紐付け |
| 4 | 不具合起票（テストケースから） | テストケースに紐づけて不具合を起票 |
| 5 | 不具合起票（単独） | テストケースと無関係に不具合を起票 |
| 6 | 不具合に資料を添付 | ファイルを検索して不具合に紐付け |
